
# ESP32-CAM 智能小车控制系统

## 项目概述
本项目基于 ESP32-CAM 开发板实现一个可通过 WiFi 控制的智能小车系统。用户通过浏览器访问控制界面，使用按钮或触摸屏控制小车的前进、后退、转向和速度调节。系统通过 UART2 与电机驱动模块通信，实现精确的电机控制。

### 关键特性
- 📱 **Web 控制界面**：响应式设计，支持移动设备触摸操作
- 🚗 **实时控制**：低延迟的电机控制响应
- 🔄 **自动停止**：300ms 无指令自动停车，提高安全性
- ⚡ **速度调节**：50-100 级可调速度控制
- 🔧 **错误检测**：内置电机驱动错误检测机制

## 前置条件
### 硬件要求
- ESP32-CAM 开发板
- 电机驱动模块（兼容 Qik 协议）
- 直流电机 x2
- 杜邦线若干
- 电源（5V-12V）

### 软件要求
- Arduino IDE 1.8.10+
- ESP32 开发环境支持
- 支持 HTML5 的现代浏览器

## 运行方法
1. **硬件连接**：
   - UART2 TX (GPIO14) → 电机驱动 RX
   - UART2 RX (GPIO15) → 电机驱动 TX
   - ERR_PIN (GPIO13) → 电机驱动错误输出
   - RET_PIN (GPIO12) → 电机驱动复位引脚

2. **烧录固件**：
   - 使用 Arduino IDE 打开 `ESP32-Cam_Car.ino`
   - 选择正确的开发板和端口
   - 上传代码

3. **连接控制**：
   - 设备会创建名为 "ESP32-CAM-AP" 的 WiFi 热点
   - 密码：12345678
   - 连接后访问 http://192.168.4.1 进入控制界面

4. **控制操作**：
   - 前进/后退：点击 Up/Down 按钮
   - 转向：点击左/右按钮
   - 速度调节：拖动速度滑块

## 构建方法
1. 克隆或下载项目代码
2. 在 Arduino IDE 中打开 `.ino` 文件
3. 安装所需库（如有）
4. 配置开发板设置：
   - 开发板：AI Thinker ESP32-CAM
   - 上传速度：921600
   - Flash 频率：40MHz
5. 编译并上传

## 技术细节
- **通信协议**：UART2 (9600, 8N1)
- **控制协议**：Qik 电机驱动协议
- **Web 服务器**：ESP32 WebServer 库
- **错误处理**：GPIO13 中断/轮询检测

## 故障排除
- 若小车无响应：
  1. 检查 UART2 连接
  2. 确认电机驱动供电
  3. 查看串口调试信息
- 若连接不上 WiFi：
  1. 重启设备
  2. 检查热点是否可见
